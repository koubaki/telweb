What is TelWeb?

TelWeb resolves domain names into telephone numbers, webfingers (Fediverse), emails, and profile information.

Features

  • Made originally for phone numbers, but can be used for any contact information.
  • Supports phone number, webfinger, email, and profile information.
  • Simple to implement.
  • DNS fallback for phone numbers only (click here for it).
  • Open source and free to use.
  • You don't have to remember phone numbers or other formats anymore.

Why Not [Insert Name]?

Why Not DNS?

It has limitations in length.

Why Not DID?

DID (including centralized ones like PLC) is cryptographic (you can lose your keys and identity) and is append-only. It's not suitable for private-ish information like phone numbers.

How To

For Clients

To resolve a domain name, simply send a request to https://[domain]/.well-known/telweb. The response will be a JSON-LD object of this type containing the resolved information.

For Servers

To implement TelWeb, create an endpoint at /.well-known/telweb that returns a JSON-LD object of this type containing the relevant information for the user.

Make sure to implement a good interface for the user to set their contact information, and let them add custom fields in their profile.

DNS Fallback

For phone numbers only, if the TelWeb endpoint is not available, clients can fallback to DNS by querying for a TXT record at _telweb.[domain]. The response should be in the format tel:[number].

Style

Phone Numbers

When referring to phone numbers, use the E.164 format.

Webfingers

When referring to webfingers, use either the standard email format, @[username]@[domain], or acct:[username]@[domain].

Emails

When referring to emails, use the standard email format [username]@[domain].

Profiles

We don't define profile information, but it must be an object.

Some common profile fields include name (string), bio (string), and avatar (URL). Expand it as you wish.
